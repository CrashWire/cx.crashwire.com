3.6 released December 5th, 2001
  • Added Breakpoint cards as well as support for the Victory keyword.
3.5 released July 27th, 2000
  • Further optimization of the card database.
  • Replaced text readme file with proper help file.
  • You can no longer increase the size of the windows beyond the initial size.
3.41 released February 9th, 2000
  • DeckAlyz now correctly calculates the double resources given if you have a Dragon, Dresdner, or Golani HQ and a deck made up from the right domain (Bion was already correct).
3.4 released February 1st, 2000
  • Added support for the new "Arctic" keyword.
  • Altered card data format to fit 2-digit deploy costs.
  • Cleaned up code to make smaller executable and require less memory.
  • Fixed URLs to my new homepage.
  • Corrected a typo in Domain HQ warnings.
3.3 released November 23rd, 1999
  • Added support for top deck cards, including warnings for excessive amounts, consideration in calculations, and a counter.
  • Due to a card change by GA making it redundant, removed code for Pillbox and Perimeter Fence corrections for updating card database.
  • Numerous minor interface tweaks.
3.2 released July 16th, 1999
  • Changed the way Rook Armor Chassis and Bull Harness's firepower and armor are calculated; now instead of the values being assigned according to the averages they are given minimum values.
  • Updated calculations of the assets sacrificable to contracts to fit the recent card changes.
  • Fixed bug counting assets sacrificable to Software Maintainance Contract.
  • Removed the "Update" button. It was more confusing than useful. The update option is still available via the menu.
  • Minor interface tweaks.
3.01 released April 4th, 1999
  • Fixed bug that caused DeckAlyz not to recognize card names under certain conditions.
  • A hint detailing the full name of a card can be seen if it is too wide to fit in the list by holding the mouse over it.
3.0 released March 8th, 1999
  • Added support for "Transcendent" and "Inner Circle" as domains instead of keywords.
  • Added control over inclusion of card text in card data updates.
  • DeckAlyz will now display an error message if it encounters unknown card names in a deck file. It will ignore these lines and process the deck without them.
  • Rewritten code so that total size of all files is nearly 300K smaller than version 2.2, and large decks should be analyzed considerably faster.
  • DeckAlyz now recognizes resource producing contracts, warning if there are not enough assets suitable to sacrifice, and adding 30 resources (2 per turn*15 turns) to the appropriate resource produced counter.
  • DeckAlyz now recognizes domain HQs, and will warn if there are other domains in the deck. It will also calculate the double cyber produced by a Bion HQ in an all-monastery deck.
  • Double Clicking on the resources produced boxes now also gives the list of appropriate contracts and HQs.
  • Rearranged card stats reader interface.
2.2 released February 17th, 1999
  • Added support for the "Spearhead", "Pilot", "Veteran" and "Unique" keywords.
  • Fixed bug where cards with long names would sometimes turn into Zapps.
  • Added chance for opening hand to contain selected card.
  • Added new copy card to clipboard functionality.
  • Added right-click popup menu to card list.
  • Added Rook Class Assault Chassis to the possible card corrections.
  • Reworded several problem identification messages.
2.13 released November 17th, 1998
  • Fixed bug with "save..." option, that would create output files that didn't conform to the format of deck files.
  • Returned the "other" counter in the enhancement row, since there is in fact one keywordless enhancement (Radiation Sickness).
2.12 released November 12th, 1998
  • Now all the windows appear in center of screen, not just the main window.
  • Focus always returns to the main window when closing the other windows.
  • Fixed the bug which caused double clicks on the first item in the card list to be treated as if no card is selected.
  • Fixed the bug which caused the card list to display only Command Programs when the problem box was double clicked.
  • Removed the 500 card-type limitation. You can now theoretically create decks with over 500 different cards.
2.11 released October 9th, 1998
  • Fixed small incompatibility with post-Ascension versions of CXlist that caused some text to be missing in several cards (like Uplift or GenSys Tech)
  • Fixed several spelling mistakes.
  • DeckAlyz now appears in center of screen regardless of resolution (minimum res 800*600).
2.1 released September 21st, 1998
  • The "Load" button has been renamed "Open".
  • Added support for the "Yakusa", "Item", "Command" and "Direct" keywords.
  • Since there are no longer keywordless enhancements, programs, or interventions, removed the "other" counters.
  • Since there are now too many asset keywords to fit in a single row, Added scroll arrows that enable you to see the various asset keywords. In addition, "AntiTerrorism" has been abbreviated to "AT".
  • Added warning in case of Yakusa assets without Yakusa Contract.
2.0 released August 8th, 1998
  • Improved "Find Text" option in Card Database reader to identify partial card names.
  • Added proper error handling in case of Invalid files.
  • Analyzing a large deck should be slightly faster.
  • Fixed "Core Collapse" bug.
  • The "Cancel" button on file dialogs now works properly.
  • Added statistics for new keywords introduced in Ascension (Debtor and Ice)
  • Added rarity statistics.
  • Redesigned the interface.
  • In event of multiple HQs, DeckAlyz will now calculate resources according to the first HQ in the file, rather than the sum of all HQs.
